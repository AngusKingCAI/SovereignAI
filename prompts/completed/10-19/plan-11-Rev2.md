S0.3 — Add new rules (revised from Rev1 per Round Table adjudication):
- **OR54**: Memory backends are pluggable components discovered via the CapabilityGraph, not hardcoded in the core. The Librarian queries the graph for backends declaring `memory_storage` and `memory_query` capabilities. Routing is by capability `name` string — any backend declaring a new memory type name (e.g. `vector`, `semantic`) is routed to automatically; the four v1 types (`episodic`, `procedural`, `trace`, `working`) are documentation of what ships, not a closed enum. Source: Plan 11 Rev2 (F-15).
- **OR55**: Each SQLite memory backend owns a separate database file: `~/.sovereignai/episodic.db`, `~/.sovereignai/trace.db`. Procedural memory uses a JSON file at `~/.sovereignai/procedural_memory.json`. Working memory is in-process only. All files are created automatically on first access via `os.makedirs(..., exist_ok=True)`. Source: Plan 11 Rev2 (F-3).
- **OR56**: Crash recovery is automatic and non-blocking. On startup, the trace backend scans for tasks whose last `TaskStateChanged` event has a non-terminal state (`RECEIVED`, `QUEUED`, `EXECUTING`). For each such task, the system emits a WARN trace, marks the task as `FAILED` in the trace log with message `"recovered: incomplete at shutdown"`, and does NOT replay side effects (side effects may not be idempotent). The user is informed via the Orchestrator panel (which reads traces), not via a blocking UI dialog. Source: Plan 11 Rev2 (F-8, F-11, F-46).
- **OR66**: All durable memory backends use atomic writes. SQLite backends use transactions (WAL mode, `busy_timeout=5000`). The JSON procedural backend writes to a temp file then `os.replace()` (atomic on Windows and POSIX). No backend may use bare `open().write()` for durable data. Source: Plan 11 Rev2 (F-2).

## Plan Body

### S1 — Create sovereignai/librarian/librarian.py

The Librarian is the memory router. It does not store data itself — it routes queries to backends based on capability.

**Constructor** (≤15 args per AR5):
```python
def __init__(
    self,
    capability_graph: CapabilityGraph,
    trace: TraceEmitter,
) -> None:
```

**Methods**:
- `store(memory_type: str, data: dict, metadata: dict = None) -> str`: Route store request to the backend declaring `memory_storage` for `memory_type`. Returns the generated record id. If multiple backends match, store to the highest-priority backend only (first-wins by priority, not scatter-gather — store is a write, not a read).
- `query(memory_type: str, query: dict) -> list[dict]`: Route query to backends declaring `memory_query` for `memory_type`. Scatter-gather if multiple backends match, with merge semantics per memory type (see below).
- `delete(memory_type: str, record_id: str) -> bool`: Route delete to the backend that owns the record.
- `_route(memory_type: str, capability: str) -> list[ComponentId]`: Query CapabilityGraph for backends declaring `memory_{capability}` with name matching `memory_type`.

**Scatter-gather merge semantics** (per Rev2 F-16):
- `episodic`: union, deduplicate by record id, sort by timestamp ascending. Returns one ordered timeline.
- `procedural`: union, deduplicate by pattern id, keep highest confidence for duplicates.
- `trace`: union, deduplicate by record id, sort by timestamp ascending.
- `working`: first-backend-wins (no merge — working memory is task-scoped, only one backend should match).
- Any future memory type: union, deduplicate by record id. Documented default; backend may override.

**Routing extensibility** (per Rev2 F-15): `_route()` queries the CapabilityGraph by capability `name` string. The four v1 types are documentation of what ships in this plan — a future backend declaring `[[capabilities]] category="memory_storage", name="vector"` will be routed to automatically without Librarian changes.

### S2 — Create sovereignai/memory/episodic_backend.py

SQLite backend for episodic memory.

**Storage**: `~/.sovereignai/episodic.db` (per Rev2 OR55 — separate file, not shared `memory.db`).

**PRAGMAs** (set on every connection open):
```python
conn.execute("PRAGMA journal_mode=WAL")
conn.execute("PRAGMA busy_timeout=5000")
```

**Schema**:
```sql
CREATE TABLE IF NOT EXISTS episodes (
    id TEXT PRIMARY KEY,           -- UUID, generated by store()
    timestamp REAL NOT NULL,
    component TEXT NOT NULL,
    task_id TEXT,
    event_type TEXT NOT NULL,
    data TEXT NOT NULL,            -- JSON
    metadata TEXT                  -- JSON
);
CREATE INDEX IF NOT EXISTS idx_episodes_task ON episodes(task_id);
CREATE INDEX IF NOT EXISTS idx_episodes_time ON episodes(timestamp);
CREATE INDEX IF NOT EXISTS idx_episodes_task_time ON episodes(task_id, timestamp);
```

**Methods**:
- `store(data: dict, metadata: dict = None) -> str`: Generate UUID, insert into episodes table, commit. Return the UUID.
- `query(query: dict) -> list[dict]`: Query by `time_range` (tuple of floats), `component`, `task_id`, `event_type`, or `task_ids` (list of strings — uses `WHERE task_id IN (?,?,...)` with parameter binding for batch efficiency). Returns list of dicts with `id`, `timestamp`, `component`, `task_id`, `event_type`, `data`, `metadata`.
- `delete(record_id: str) -> bool`: Delete by id. Return True if a row was deleted.

**Manifest**: `adapters/internal/episodic_memory/manifest.toml`

### S3 — Create sovereignai/memory/procedural_backend.py

JSON file backend for procedural memory. Keeps JSON (not SQLite) to validate the pluggable-storage claim — a single-user app benefits from at least one file-based backend to prove the contract is storage-agnostic.

**Storage**: `~/.sovereignai/procedural_memory.json`

**Atomic write** (per Rev2 OR66):
```python
import os, json, tempfile
def _atomic_write(self, path: str, data: dict) -> None:
    """Write data to path atomically via temp file + os.replace."""
    fd, tmp = tempfile.mkstemp(dir=os.path.dirname(path), suffix=".tmp")
    try:
        with os.fdopen(fd, "w", encoding="utf-8") as f:
            json.dump(data, f, indent=2)
        os.replace(tmp, path)  # Atomic on Windows and POSIX
    except Exception:
        os.unlink(tmp)  # Best-effort cleanup
        raise
```

**File lock** (per Rev2 F-2): use a simple lock file (`~/.sovereignai/procedural_memory.json.lock`) with `os.O_EXCL | os.O_CREAT` to acquire, `os.unlink()` to release. If the lock is held for >5 seconds, force-acquire and log a WARN trace. This is not a real mutex but is sufficient for a single-user, single-process app.

**Schema**:
```json
{
  "schema_version": 1,
  "patterns": [
    {
      "id": "uuid",
      "pattern": "...",
      "confidence": 0.85,
      "last_used": "iso_timestamp",
      "success_count": 12,
      "failure_count": 2
    }
  ]
}
```

**Methods**:
- `store(data: dict, metadata: dict = None) -> str`: Generate UUID, append to patterns list, atomic-write file. Return UUID. If `len(patterns) > MAX_PATTERNS` (default 10,000), call `prune_low_confidence(0.3)` before storing.
- `query(query: dict) -> list[dict]`: Filter by `pattern_match` (substring), `min_confidence` (float), `last_used_after` (ISO timestamp). Returns matching patterns.
- `delete(record_id: str) -> bool`: Remove by id, atomic-write file. Return True if removed.
- `prune_low_confidence(threshold: float = 0.3) -> int`: Remove all patterns with confidence < threshold. Return count removed.

**Corruption recovery** (per Rev2 F-2): if `json.load()` raises `JSONDecodeError` on startup, rename the corrupted file to `procedural_memory.json.corrupted.{timestamp}`, start with an empty pattern list, and emit a WARN trace: `"procedural_memory.json was corrupted; backed up to {path}; starting fresh"`. This satisfies P9 (no silent failures) — the user is informed, and the corrupted file is preserved for manual recovery.

**Manifest**: `adapters/internal/procedural_memory/manifest.toml`

### S4 — Create sovereignai/memory/working_backend.py

In-memory backend for working memory.

**Storage**: Python dict, task-scoped.

**Methods**:
- `store(data: dict, metadata: dict = None) -> str`: Generate UUID, store in dict under `{task_id: {record_id: entry}}`. Return UUID.
- `query(query: dict) -> list[dict]`: Lookup by `task_id` or `record_id`. Returns matching entries.
- `delete(record_id: str) -> bool`: Remove entry by id. Return True if removed.
- `cleanup(task_id: str) -> None`: Remove all entries for task_id.

**Wiring** (per Rev2 F-7 — this was missing in Rev1): `build_container()` in S6 subscribes `WorkingMemoryBackend.cleanup` to `TASK_STATE_CHANNEL` events. On `TaskStateChanged` with `new_state in {COMPLETE, FAILED}`, call `working_backend.cleanup(event.task_id)`. This is wired in `main.py` S6, not in the backend itself.

**Manifest**: `adapters/internal/working_memory/manifest.toml`

### S5 — Create sovereignai/memory/trace_backend.py

SQLite backend for trace memory (durable version of TraceEmitter's in-memory storage).

**Storage**: `~/.sovereignai/trace.db` (per Rev2 OR55 — separate file, not shared `memory.db`).

**PRAGMAs**: same as episodic (WAL, busy_timeout=5000).

**Schema** (per Rev2 F-8 — `trace_id` column renamed to `correlation_id` to match the in-memory `TraceEvent` dataclass):
```sql
CREATE TABLE IF NOT EXISTS traces (
    id TEXT PRIMARY KEY,               -- UUID
    timestamp REAL NOT NULL,
    level TEXT NOT NULL,               -- "trace"|"debug"|"info"|"warn"|"error"
    component TEXT NOT NULL,
    message TEXT NOT NULL,
    correlation_id TEXT NOT NULL,      -- UUID grouping events from same task
    task_id TEXT,                      -- NULL if not task-correlated
    task_state TEXT                    -- NULL; set to "received"|"queued"|"executing"|"complete"|"failed" if this trace records a TaskStateChanged event
);
CREATE INDEX IF NOT EXISTS idx_traces_correlation ON traces(correlation_id);
CREATE INDEX IF NOT EXISTS idx_traces_time ON traces(timestamp);
CREATE INDEX IF NOT EXISTS idx_traces_task ON traces(task_id);
```

**Integration with TraceEmitter**: The trace backend subscribes to the TraceEmitter (via a callback registered in `build_container()`). Every `TraceEmitter.emit()` call persists the event to the `traces` table. The `task_id` and `task_state` fields are populated by inspecting the `correlation_id` — if the correlation_id matches a known task, the backend looks up the task's current state from the `TaskStateMachine` (injected) and stores it.

**`emit()` call signature** (per Rev2 F-8): The existing `TraceEmitter.emit()` signature is `emit(component: str, level: TraceLevel, message: str, correlation_id: UUID | None = None)`. The trace backend's subscriber callback receives the `TraceEvent` dataclass (which has `component`, `level`, `message`, `timestamp`, `correlation_id`). All trace-backend persistence uses the `TraceEvent` fields, not positional args. The Rev1 plan's `trace.emit(TraceLevel.WARN, "recovery", ...)` was wrong (args swapped) and is removed in Rev2.

**Methods**:
- `store(event: TraceEvent, task_id: str = None, task_state: str = None) -> str`: Insert into traces table. Return the generated id.
- `query(query: dict) -> list[dict]`: Query by `level`, `component`, `correlation_id`, `task_id`, `time_range`.
- `delete(record_id: str) -> bool`: Delete by id.
- `get_last_task_states() -> dict[str, str]`: Scan traces for the most recent `task_state` value per `task_id`. Returns `{task_id: state}`. Used by crash recovery (S7).

**Manifest**: `adapters/internal/trace_memory/manifest.toml`

Per P1 (core is sacred), this requires justification: the trace backend needs to persist every trace event durably, and the only way to observe emits without changing the `emit()` call signature is via a subscriber callback. This is a minimal, well-justified core edit. Add `sovereignai/shared/trace_emitter.py` to "Files WILL Edit."

### S7 — Crash recovery (non-blocking, automatic)

On startup (in `main.py`, after `build_container()` but before the web server starts):
```python
# Crash recovery — non-blocking, automatic (per Rev2 OR56)
trace_backend = container.retrieve(TraceMemoryBackend)
last_task_states = trace_backend.get_last_task_states()
state_machine = container.retrieve(TaskStateMachine)

for task_id_str, state in last_task_states.items():
    if state in ("received", "queued", "executing"):
        # Task was incomplete at shutdown — mark as failed, do NOT replay
        trace.emit(
            component="recovery",
            level=TraceLevel.WARN,
            message=f"Task {task_id_str} was incomplete (state={state}) at shutdown; marking as recovered-failed. Side effects not replayed.",
            correlation_id=UUID(task_id_str) if _is_valid_uuid(task_id_str) else None,
        )
        # Persist a synthetic FAILED trace so the task shows as terminal in future scans
        trace_backend.store(
            event=TraceEvent(
                component="recovery",
                level=TraceLevel.WARN,
                message=f"recovered: incomplete at shutdown (was {state})",
                timestamp=now_utc(),
                correlation_id=UUID(task_id_str) if _is_valid_uuid(task_id_str) else new_correlation_id(),
            ),
            task_id=task_id_str,
            task_state="failed",
        )
```

**No UI dialog.** The user is informed via the Orchestrator panel (which already reads traces) and the Log drawer. The web UI (Plan 8) requires no changes for this — it already renders WARN traces. A future plan may add a "Recovery" banner UI, but that is out of scope for Plan 11.

**`_is_valid_uuid` helper**: a one-liner `try: UUID(s); return True; except ValueError: return False`. Lives in `sovereignai/shared/types.py` as a new function (not a new class — minimal core addition).
