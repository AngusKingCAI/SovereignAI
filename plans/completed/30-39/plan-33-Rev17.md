## S2 — Lifecycle Hooks

S2.1: Create `app/sovereignai/lifecycle/hooks.py` — `LifecycleHookRegistry`; DI-injected, owned by LifecycleComposer
S2.2: Phases: `REGISTRATION_OPEN → REGISTRATION_CLOSED → STARTUP_RUNNING → STARTUP_DONE → SHUTDOWN_RUNNING → SHUTDOWN_DONE`. Gate: `LifecycleManager.start()` transitions `REGISTRATION_OPEN → CLOSED`. Registering after `CLOSED` raises `LifecycleError`.
S2.3: Hook failure behavior: **Critical SHUTDOWN hook failure**: log + emit `lifecycle.hook.failed` + mark stage FAILED, continue remaining teardown stages within timeouts, write sentinel before force-kill. **Non-critical hook failure** (startup or shutdown): log + emit + continue next hook. **Critical STARTUP hook failure**: emit `lifecycle.hook.failed`; abort startup with `LifecycleError` raised out of `LifecycleManager.run_startup()`. Caller (S5.1 step 6) catches: degraded-start permitted only if all critical startup hooks passed. Non-critical startup hook failure: log + emit + continue. All hook failures emit to stderr fallback and EventBus if available.
S2.4: **Canonical hook timeout rules** (supersedes all other refs): per-startup hook 15s each (sequential); per-shutdown: non-critical concurrent 15s shared + critical sequential 15s = 30s total. **Implementation**: `asyncio.wait_for(coro, 15)`; `TimeoutError` caught and counted as hook failure per S2.3. Cross-ref: S4.3 force-kill budget accounts for 30s.
S2.5: **v1 hook cap**: max 5 total startup hooks (critical + non-critical combined). Max 5 total shutdown hooks (4 non-critical + 1 critical). AR check `check_max_shutdown_hooks.py` enforces both caps.
S2.6: Plan 34 registers: startup hook = `MemoryGateway.load()` (critical=False — system starts without memory, routes return 503 until loaded), shutdown hook = `MemoryGateway.flush()` (critical=True). **Load timeout**: if `load()` exceeds 15s hook timeout, log WARN, emit `lifecycle.memory_load_timeout`, continue with memory disabled. `MemoryGateway.is_ready() -> bool`: True after `load()` completes, False after `flush()` or if load not yet completed.

## S4 — Graceful Shutdown

S4.1: SIGTERM/SIGINT handler: set state SHUTTING_DOWN. **Immediately**: (a) set `ready = false` (S3.7a), (b) emit `lifecycle.shutting_down` via EventBus with `{timestamp, server_pid, instance_uuid, drain_timeout_seconds: 60}`. Hook failures do not prevent this emission. If EventBus unavailable: emit to stderr fallback: `'[LIFECYCLE] shutting_down pid={pid} uuid={uuid}'`. TUI (Plan 32 S4.4) detects shutdown via multiple signals (SSE lifecycle.shutting_down, ready:false, sentinel, PID/UUID change). EventBus-unavailable shutdown falls back to non-SSE paths.
S4.2: Drain in-flight: 60s timeout. **Drain-time route policy — Consolidated exemption list**: `/api/health` (cached results), `/api/lifecycle/ready` (returns ready:false), `/api/lifecycle/stream` (existing connections continue; NEW connections → HTTP 503 with `Retry-After: 60`), `/api/auth/logout`, `/api/trace/logs`. **All other routes**: return HTTP 503 with `Retry-After: 60`. **Middleware ordering**: (1) Authentication/CSRF middleware runs first for all routes. (2) Drain middleware runs second — for non-exempt routes, returns HTTP 503 with `Retry-After: 60`. (3) Business handler runs only for exempt routes that passed auth. Auth failure → standard auth error code (NOT 503). This ensures `/api/auth/logout` with invalid session returns 401, not 503.
S4.3: Shutdown sequence (sequential, after drain): core services — OptionsBackend (20s) → Orchestrator (20s) → ModelRegistry (20s) → HookRegistry shutdown hooks (30s, including MemoryGateway.flush() as critical hook — Plan 34 registered) → emit `lifecycle.stopped` event → post-stopped drain (2s) → EventBus shutdown (20s). HookRegistry shutdown hooks execute BEFORE EventBus shutdown. Critical hooks (like MemoryGateway.flush()) can still emit lifecycle events through the running EventBus. Post-EventBus shutdown → no more event emission possible. **Force-kill: 190s** = 60s drain + OptionsBackend(20s) + Orchestrator(20s) + ModelRegistry(20s) + HookRegistry(30s) + EventBus(20s) + 20s headroom. `lifecycle.stopped` emitted after core services and hooks complete, before EventBus shutdown; 2s post-stopped drain folded into headroom. Shutdown timeout constants defined in `app/sovereignai/lifecycle/manager.py`:
    `DRAIN_TIMEOUT = 60`
    `OPTIONS_BACKEND_SHUTDOWN_TIMEOUT = 20`
    `ORCHESTRATOR_SHUTDOWN_TIMEOUT = 20`
    `MODEL_REGISTRY_SHUTDOWN_TIMEOUT = 20`
    `HOOK_REGISTRY_SHUTDOWN_TIMEOUT = 30`
    `POST_STOPPED_DRAIN = 2`
    `EVENTBUS_SHUTDOWN_TIMEOUT = 20`
    `FORCE_KILL_HEADROOM = 20`
    `FORCE_KILL_BUDGET = DRAIN_TIMEOUT + OPTIONS_BACKEND_SHUTDOWN_TIMEOUT + ORCHESTRATOR_SHUTDOWN_TIMEOUT + MODEL_REGISTRY_SHUTDOWN_TIMEOUT + HOOK_REGISTRY_SHUTDOWN_TIMEOUT + EVENTBUS_SHUTDOWN_TIMEOUT + FORCE_KILL_HEADROOM  # 190s`
S4.4: TUI NOT in shutdown sequence — monitors SSE `/api/lifecycle/stream` and file sentinel; exits client-side.
S4.5: On SIGTERM/SIGINT, immediately set `ready=false`, emit `lifecycle.shutting_down`, write sentinel, then begin drain sequence. This ensures TUI detects shutdown even if graceful exit completes before force-kill. Write sentinel at `platformdirs.user_cache_dir("sovereignai") / f"shutdown.{os.getpid()}.sentinel"` — atomic write, fsync. Sentinel body: `{datetime.now(timezone.utc).isoformat()}\n{instance_uuid}\n`. Format MUST match Plan 32 S4.4 (TUI parser contract). Emit `lifecycle.shutdown.forced` if EventBus available; stderr fallback otherwise. **Sentinel cleanup**: On startup, read existing sentinel file. Check PID liveness: os.kill(pid, 0). If ProcessLookupError (PID dead) AND sentinel age > 5 minutes → delete sentinel. If PID alive → another process is running → STOP with error. Do NOT use current-process UUID to decide staleness for a different PID's sentinel. UUID validation remains in Plan 32's matching live-process shutdown signal only.

## S6 — Circuit Breaker Integration

S6.1: Worker circuit breaker per AR7: >50 errors in 10s sliding window (event-count resets per 10s bucket) per worker (`worker_id`) = unload. Window mechanism: sliding window — count errors with timestamp > now - 10s. Error includes timeout (>15s without response). After unload: worker quarantined 60s; dispatch to that worker_id rejected with `worker_unavailable`; after cooldown, re-eligible for load.
S6.2: Orchestrator circuit breaker: 10 consecutive message failures within consecutive-failure-with-60s-gap-reset = DEGRADED. Any success resets the run. If >60s since last failure, run resets. **Success criterion**: `OrchestratorResponse` with `error is None`. **Failure**: exception raised (any), validation error, OR response unparseable. A 200 response with clarification content in `response_text` but `error is None` counts as SUCCESS for circuit breaker purposes — the Orchestrator successfully processed the request and returned a valid business response. **Half-open probe definition**: After 60s cooldown, invoke Orchestrator.health_check() — a lightweight no-op method on the Orchestrator interface that returns OrchestratorResponse(error=None, state="idle") without processing any message. This validates that the Orchestrator is responsive without side effects. Add test: test_half_open_probe_succeeds_with_health_check, test_half_open_probe_failure_resets_to_open, test_clarification_request_with_error_none_counts_as_success. **Recovery from OPEN**: After 60s cooldown, enter HALF_OPEN. Execute health_check() probe. If probe succeeds (error is None) → increment recovery counter. If probe fails → reset to OPEN, restart cooldown. 3 consecutive successful probes → CLOSED. Define "HEALTHY" in recovery context as "probe returned error=None". For S6.3 adapter circuit breakers, separate adapter health status from circuit-breaker state — recovery defined by 3 consecutive successful adapter-specific health checks, not Orchestrator health. Open-state behavior: all dispatches to the Orchestrator return `service_degraded` error (no timeout wait). Concurrent half-open probes restricted to exactly one.
S6.3: Adapter circuit breaker: `health_check()` fails 3x = UNHEALTHY; recovery: 60s cooldown, half-open probe, 3 consecutive HEALTHY polls → close.
S6.4: All circuit events logged via TraceEmitter per AR8; stderr fallback if EventBus unavailable.
